# ピッチ DSL（度数・コード）

MIDI シーケンスの `play()` には、スライス番号の代わりに**度数（degree）**を書きます。度数とは「トニック（根音）からの音程」を数字で表したものです。たとえば `global.key("C")` なら度数 `1` は C、度数 `3` は E になります。

## 基本的な度数の書き方

```text
var global = init GLOBAL
global.tempo(120)
global.beat(4 by 4)
global.key("C")
global.start()

var piano = init global.seq
piano.midi("IAC", 1)
piano.octave(4)
piano.vel(96)
piano.length(1)
piano.play(1, 3, 5, 1)   // C4 E4 G4 C4

LOOP(piano)
```

使える度数は **1〜9, 11, 13** です。10, 12, 14 以上は使えません（オクターブは後述の `^N` で表します）。

| 度数 | C major での音 |
|---|---|
| `1` | C |
| `2` | D |
| `3` | E |
| `4` | F |
| `5` | G |
| `6` | A |
| `7` | B |
| `8` | C（1 のオクターブ上と同じ） |
| `9` | D（1 の長 9 度上） |
| `0` | 休符 |

### 変音記号

フラット `b` とシャープ `#` を度数の前に付けられます。

```text
piano.play(1, b3, 5, b7)   // 短 3 度・短 7 度（マイナー7thコードの音）
piano.play(1, 3, #5, 7)    // 増 5 度
```

## `^N` — オクターブシフト（スティッキー）

`^N` を音の直後に付けると、その音から「N オクターブ上（または下）にシフト」します。このシフトは**持続する（スティッキー）**ので、次の音にも引き継がれます。

```text
piano.play(1, 3, 5^+1, 7)
// 1=C4, 3=E4, 5^+1=G5(+1 oct), 7=B5(前の^+1が残る)
```

基準に戻したいときは `^0` を書きます。

```text
piano.play(1^+1, 3, 5, 1^0, 3, 5)
// 1^+1=C5, 3=E5, 5=G5, 1^0=C4(^0でリセット), 3=E4, 5=G4
```

下にシフトするときはマイナスを付けます。

```text
piano.play(1^-1, 3^-1, 5^-1)   // 1オクターブ下
```

## `[ ]` — コードスタック（同時発音）

`[ ]` の中に複数の度数を書くと、その拍に**和音（コード）**が鳴ります。

```text
piano.play([1, 3, 5])         // Cトライアド（C E G）
piano.play([1, 3, 5, 7])      // Cmaj7 コード
piano.play([1, b3, 5, b7])    // Cm7 コード
```

複数のコードを並べてコード進行も作れます。

```text
piano.play([1, 3, 5], [5, 7, 2], [6, 8, 3], [4, 6, 8])
// Cトライアド → Gトライアド → Am → F
```

### コードを変数に入れる

よく使うコードを変数に入れておくと再利用できます。

```text
var m7 = [1, b3, 5, b7]     // マイナー7th コード

piano.play(m7, 0, m7, 0)    // Cm7・休・Cm7・休
```

コード変数は `[ ]` 記法なので「縦（同時発音）の値」です。`( )` の変数（後述のパターン変数）とは区別されます。

### スプレッドと音の追加・削除

```text
var m7    = [1, b3, 5, b7]
var m7add9 = [m7, 9]         // m7 に 9th を追加
var m7omit5 = [m7, -5]       // m7 の 5 度を除く
```

### stdlib のコード

```text
import chords

piano.play(maj7, m7, dom7)   // Cmaj7, Cm7, C7
```

`import chords` で標準ライブラリの定義済みコード（`m7`, `maj7`, `dom7`, `m7b5`, `dim7`, `sus4` など）が使えるようになります。

## `*n` — 繰り返し

`*n` を付けると、その要素を **n スロット分** 続けて配置します。

```text
var riff = (1, 5, 8, 5)

bass.play(riff*2)   // riff を 2 回繰り返した長さになる（4 スロット × 2 = 8 スロット）
```

::: warning Tidal の `*` とは意味が異なります
OrbitScore の `*n` はそのグループを **n スロット分占有**します（Tidal の `!` に相当）。
Tidal の `*n`（1 スロット内を n 等分）ではありません。1 スロット内の繰り返しは `(1, 1)` のようにネストで書きます。
:::

## パターン変数とセクション変数

`( )` で囲んだパターンを変数に代入してリストアップし、楽曲の構造を組み立てられます。

```text
var riff = (1, 5, 8, 5)          // パターン変数（1セル）
var sec  = (1, 3, 5), (5, 3, 1)  // セクション変数（カンマ区切り = 2セル）

bass.play(riff*2)                // riff を 2 回繰り返す
lead.play(sec, sec)              // sec の 2 セルをまた 2 回並べる（AABA 形式なども同様）
```

変数を使い回すことで、A・B・C など楽曲の各セクションを整理して書けます。

::: tip 変数の評価タイミング
変数は `play()` が評価されたときに展開されます。変数を再定義しても、すでに動いているパターンには影響しません。パターンを変えたいときは `play()` をもう一度実行します。
:::

## `{ }` — レガート（スラー）グループ

`{ }` で囲んだグループの音は、次の音と少し重なって発音します（スラー効果）。

```text
piano.play({1, 3, 5, 8})   // 4 音が滑らかにつながる
```

---

## `_` — タイ（音の伸ばし）

`_` を単体で書くと、直前の音を 1 スロット分伸ばします（リトリガーなし）。

```text
piano.play(1, _, 3)   // 1 を 2 スロット分伸ばしてから 3
```

コードスタック全体を伸ばすにはコードの直後に `_` を置きます。

```text
piano.play([1, 3, 5], _)   // Cトライアドを 2 スロット分伸ばす
```

---

## `@v` と `@g` — ノート単位の表情

音符ごとにベロシティやゲートを変えたいときは `@v`・`@g` 修飾子を使います。

```text
piano.play(5@v110, 3@v60, 1@v80)   // 各音のベロシティを絶対値で指定
piano.play(5@v+20, 1@v-10)         // シーケンスの vel() に対する相対値
piano.play(5@g30, 1@g100)          // ゲートをパーセント指定（30 = 0.30 = スタッカート）
```

`@v` と `@g` を組み合わせることもできます。

```text
piano.play(5@v110@g30)   // 強音 + スタッカート
```
