# パターンを作る

[はじめての音](../getting-started/first-sound.md) では `drum.play(1, 1, 1, 1)` と書いて、4 拍すべてに音を鳴らしました。この章では `play()` が実際に何をしているのかを説明し、休符を入れたり、拍を細かく分割したりする方法を紹介します。

## play() の基本

`play()` の引数に並べた数値が、各拍でどのスライス（音のかけら）を鳴らすかを決めます。

数値の意味は次のとおりです:

| 値 | 動作 |
|---|---|
| `0` | 休符（その拍は鳴らさない） |
| `1` 以上の整数 | 鳴らすスライスの番号 |

スライスとは、`chop()` で音声ファイルを分割したときの各ピースのことです。`chop(1)` または `chop()` を省略した場合は「音声ファイル全体がスライス 1」なので、`1` と書けばそのファイル全体を再生し、`0` と書けば無音になります。ドラムのサンプル音源を鳴らすときは、この状態がほとんどです。

```text
var global = init GLOBAL
global.tempo(120)
global.beat(4 by 4)
global.audioPath("./audio")
global.start()

var drum = init global.seq
drum.audio("kick.wav")
drum.play(1, 1, 1, 1)  // 4 拍すべてで kick.wav を鳴らす

LOOP(drum)
```

## 休符を入れてパターンを作る

`0` を入れると、その拍は無音になります。

```text
drum.play(1, 0, 1, 0)  // 1 拍目と 3 拍目だけ鳴らす
```

この例では 2 拍目と 4 拍目が休符になります。キックとスネアを組み合わせた基本的なビートは次のように書きます。

```text
var global = init GLOBAL
global.tempo(120)
global.beat(4 by 4)
global.audioPath("./audio")
global.start()

var kick = init global.seq
kick.audio("kick.wav")
kick.play(1, 0, 1, 0)  // 1・3 拍目に kick

var snare = init global.seq
snare.audio("snare.wav")
snare.play(0, 1, 0, 1)  // 2・4 拍目に snare

LOOP(kick, snare)
```

`play()` に並べる数値の個数がそのまま「そのシーケンスが 1 ループ中に刻む拍数」になります。4 つ並べれば 4 分割、8 つ並べれば 8 分割です。

## 8 分音符パターン

引数を 8 個にすると、1 小節を 8 等分した 8 分音符単位のパターンになります。

```text
var hihat = init global.seq
hihat.audio("hihat.wav")
hihat.play(1, 1, 1, 1, 1, 1, 1, 1)  // 8 分音符でハイハットを刻む
```

`0` を混ぜれば、細かい休符も自由に作れます。

```text
hihat.play(1, 1, 1, 0, 1, 1, 1, 0)  // 4 番目と 8 番目の 8 分音符を抜く
```

## length() でパターン全体の長さを変える

`length()` はパターン全体が何小節ぶんかを設定します。省略するとデフォルトは 1 小節です。

```text
seq.length(1)   // 1 小節ループ（デフォルト）
seq.length(2)   // 2 小節ループ
seq.length(4)   // 4 小節ループ
```

2 小節のパターンを書くには、`play()` の引数を 2 小節ぶん並べます。

```text
var global = init GLOBAL
global.tempo(120)
global.beat(4 by 4)
global.audioPath("./audio")
global.start()

var kick = init global.seq
kick.audio("kick.wav")
kick.length(2)
kick.play(
  1, 0, 0, 0,   // 1 小節目
  1, 0, 1, 0,   // 2 小節目（少し変化をつける）
)

LOOP(kick)
```

::: warning length() は音程にも影響します
`length()` を変えると、1 イベントあたりに割り当てられる時間が変わります。`chop()` でファイルを分割して使っている場合、再生速度が変わって音程も変化します。たとえば `length(2)` にすると再生速度が半分になり、音程が 1 オクターブ下がります。`chop(1)`（デフォルト）でファイル全体を使うドラムサンプルでは通常問題になりませんが、ループ素材を細かく分割して使うときは注意してください。詳しくは [オーディオ操作](./audio-manipulation.md) で説明します。
:::

## beat() で拍子を設定する

`beat()` はシーケンスの拍子（タイムシグネチャ）を設定するメソッドです。`play()` とは別の役割を持っているので、混同しないよう注意してください。

```text
global.beat(4 by 4)   // グローバルを 4/4 拍子に設定
global.beat(3 by 4)   // グローバルを 3/4 拍子に設定
global.beat(5 by 4)   // グローバルを 5/4 拍子に設定
```

シーケンスに個別の拍子を設定して、グローバルとは異なる拍子で走らせることもできます。この使い方はポリメーターの章で詳しく扱います。

## ネストしたリズム表現

括弧 `()` を使うと、1 つの拍をさらに細かく分割できます。

```text
drum.play(1, (1, 1), 0, 1)
// 1 拍目: kick
// 2 拍目: kick を 2 回（その拍を 2 等分して 2 打）
// 3 拍目: 休符
// 4 拍目: kick
```

括弧の中に書いた要素の数が、その拍をさらに何等分するかを決めます。3 つ書けば 3 等分（3 連符）、4 つ書けば 4 等分です。

```text
// 4 拍目を 3 連符にする例
drum.play(1, 0, 1, (1, 0, 1))
```

ネストは複数段にすることもできますが、深くなりすぎると読みにくくなるため、2〜3 段程度にとどめることをお勧めします。ネストを使った複雑なリズムの例は `examples/04_nested_rhythms.orbs` に掲載されています。

## まとめ

- `play()` の引数はスライス番号（`0` は休符、`1` 以上は鳴らすスライス番号）
- 引数の個数がそのまま 1 ループ中の拍数になる
- `length()` でパターン全体の小節数を変えられる（音程にも影響するため注意）
- `beat()` は拍子の設定であり、リズムパターンの定義ではない
- 括弧 `()` でネストすると 1 拍をさらに細かく分割できる
