📚 Overview

This guide provides comprehensive testing procedures for OrbitScore's audio-based live coding DSL. Tests verify implemented features against the specification in ../core/INSTRUCTION_ORBITSCORE_DSL.md.

🎯 Prerequisites

System Requirements

  • Node.js v22.0.0+
  • SuperCollider (audio engine)
  • VS Code / Cursor / Claude Code

macOS:

brew install --cask supercollider

Linux:

sudo apt-get install supercollider

Windows:

  • Download from SuperCollider official site

cd packages/engine
npm install

ls -la packages/engine/supercollider/synthdefs/

ls -la test-assets/audio/
# Verify: kick.wav, snare.wav, hihat.wav, bass.wav, arpeggio_c.wav

Testing Procedures

Test 1: Syntax Highlighting

  1. Open examples/09_reserved_keywords.orbs
  2. Verify:
    • Keywords (var, init, GLOBAL, RUN, LOOP, MUTE) are highlighted
    • Method names (tempo, beat, play, audio) are highlighted
    • Comments are properly colored

Test 2: Autocomplete

  1. Create a new line in the file
  2. Type global. → Verify: Suggestions include tempo(), beat(), start(), stop()
  3. Type seq. → Verify: Suggestions include audio(), play(), chop(), length()

Test 3: Command Execution (Cmd+Enter / Ctrl+Enter)

Note: This tests the core live coding workflow.

Step 1: Initialize

var global = init GLOBAL
global.tempo(120)
global.beat(4 by 4)
global.audioPath("../test-assets/audio")
global.start()
  • Select all lines
  • Press Cmd+Enter (Mac) or Ctrl+Enter (Linux/Windows)
  • Expected: No errors in output panel

Step 2: Create Kick Sequence

var kick = init global.seq
kick.audio("kick.wav")
kick.play(1, 0, 1, 0)
  • Select lines → Cmd+Enter
  • Expected: Sequence defined (no sound yet)

Step 3: Run Kick (Reserved Keyword)

RUN(kick)
  • Select line → Cmd+Enter
  • Expected: Kick plays once on beats 1 and 3

Step 4: Loop Kick

LOOP(kick)
  • Select line → Cmd+Enter
  • Expected: Kick loops continuously

Step 5: Mute Kick

MUTE(kick)
  • Select line → Cmd+Enter
  • Expected: Kick loops but produces no sound

Step 6: Unmute (Unidirectional Toggle)

MUTE()
  • Select line → Cmd+Enter
  • Expected: Kick unmutes (sound resumes)

Step 7: Stop

global.stop()
  • Select line → Cmd+Enter
  • Expected: All sequences stop

🔍 Feature Verification Checklist

Core DSL Features (Implemented)

Initialization

  • var global = init GLOBAL
  • var seq = init global.seq

Global Parameters

  • global.tempo(120) - Set BPM
  • global.beat(4 by 4) - Set time signature
  • global.audioPath("path") - Set audio base directory
  • global.start() - Start scheduler
  • global.stop() - Stop scheduler

Sequence Configuration

  • seq.audio("file.wav") - Load audio file
  • seq.chop(n) - Divide into n slices
  • seq.play(1, 0, 1, 0) - Set play pattern
  • seq.beat(4 by 4) - Sequence-specific time signature
  • seq.length(2) - Loop length in bars
  • seq.tempo(140) - Sequence-specific tempo

Transport Control - Reserved Keywords (DSL v3.0)

  • RUN(seq1, seq2) - One-shot playback
  • LOOP(seq1, seq2) - Loop playback (others auto-stop)
  • MUTE(seq1) - Mute flag (LOOP only, not RUN)

Unidirectional Toggle Behavior:

  • RUN/LOOP independence (same sequence can be in both)
  • MUTE persistence (flag maintained across LOOP changes)
  • Auto-stop (sequences not in LOOP are stopped)
  • Auto-unmute (sequences not in MUTE are unmuted)

Underscore Prefix Pattern (DSL v3.0)

  • _audio(), _chop(), _play() - Immediate application
  • _tempo(), _beat(), _length() - Immediate application
  • global._tempo(), global._beat() - Immediate for inheriting sequences

Gain & Pan Control

  • seq.gain(-6) - Real-time gain (dB)
  • seq.pan(-50) - Real-time pan (-100 to 100)
  • seq.defaultGain(-3) - Initial gain (before playback)
  • seq.defaultPan(20) - Initial pan (before playback)

Method Chaining

  • All methods return this for chaining

Audio Engine (SuperCollider Integration)

  • WAV file support
  • Audio slicing (chop(n))
  • 0-2ms latency
  • 48kHz/24bit output
  • Buffer preloading and caching

Not Yet Implemented

  • fixpitch() - Pitch shift
  • time() - Time stretch
  • offset() - Start position adjustment
  • reverse() - Reverse playback
  • fade() - Fade in/out
  • Composite meters - ((3 by 4)(2 by 4))
  • MIDI support
  • DAW Plugin (VST/AU)

🐛 Troubleshooting

Issue: No Sound

Check:

  1. SuperCollider is running
  2. Audio files exist in test-assets/audio/
  3. System volume is up
  4. Correct audio device selected

Solution:

# Verify audio files
ls -la test-assets/audio/*.wav

# Check engine logs
# Look for "Failed to load" or "File not found" messages

Issue: Cmd+Enter Does Nothing

Check:

  1. File has .orbs extension
  2. Extension is activated (restart IDE)
  3. Keybinding is registered

Solution:

  • Restart IDE
  • Use Command Palette: "OrbitScore: Run Selection"

Issue: Parser Error

Common Errors:

  • "Unknown character": Check comment formatting
  • "Unexpected token": Verify syntax (matching parentheses)
